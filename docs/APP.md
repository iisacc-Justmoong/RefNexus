PureRef는 레퍼런스 이미지를 한곳에 모아 무한 캔버스에 배치하고, 필요 시 창을 “항상 위”로 고정해 작업 화면 위에 얹어두는 방식으로 영감을 즉시 참조하게 만드는 경량 도구로 알려져 있으며, 최소한의 인터페이스를 유지한 채 이미지 정렬·배치, 노트 작성, 디새츄레이션 토글 같은 실무 기능을 제공하고 Windows·macOS·Linux를 지원한다. 또한 작업 상황에 따라 클릭을 아래 창으로 통과시키는 모드(Transparent to mouse)나 선택한 이미지를 떠다니는 오버레이 형태로 만드는 모드(Overlay selection), 그리고 장면을 .pur로 저장·불러오거나 캔버스 영역을 단일 이미지로 내보내는 흐름이 핵심 사용성으로 자리 잡아 있다. ([PureRef][1])

이 전제를 그대로 계승하면서 “상위 호환”을 목표로 한 가상 제품을 **RefNexus**라 정의하며, 제품 정체성은 “PureRef급 즉시성(드래그 앤 드롭, 상시 상단, 가벼움)을 1차 목표로 고정하되, 대형 보드·다수 프로젝트·검색·버전 관리·출처 관리까지 한 단계 위로 끌어올린 오프라인 레퍼런스 운영체제”로 설정한다. 서버는 전혀 사용하지 않으며, 로그인·동기화·원격 저장을 기본 기능에서 제거하고, 모든 데이터는 로컬 파일과 로컬 데이터베이스로만 관리하는 것을 원칙으로 둔다.

호환성의 설계는 “행동 호환”과 “데이터 호환”으로 분리한다. 행동 호환은 PureRef의 대표 사용 패턴을 그대로 재현하는 것으로, 항상 위 고정, 투명 클릭-스루, 오버레이 선택, 이미지/노트의 이동·회전·스케일, 정렬·정돈, 디새츄레이션 토글을 동일한 수준의 마찰로 제공하고 단축키 프리셋에서 PureRef 스타일을 선택하면 체감이 즉시 같아지도록 만든다. 데이터 호환은 .pur 이식이 핵심인데, PureRef 2.0부터 .pur 포맷이 성능 및 기능 확장을 위해 변경되었으므로, RefNexus는 “가급적 직접 임포트”를 목표로 하되 포맷이 비공개이거나 향후 변경될 위험을 상정하여, PureRef가 제공하는 “장면(캔버스) 단일 이미지 내보내기” 및 “선택 이미지 내보내기” 경로를 이용한 마이그레이션 마법사도 기본 탑재한다. 즉 최선은 레이아웃과 메타데이터까지 보존하는 직접 임포트이고, 차선은 PureRef의 내보내기를 이용해 레퍼런스를 손실 없이 이주하되 레이아웃은 RefNexus에서 자동 재구성하는 경로로 설계한다. ([PureRef][2])

RefNexus가 PureRef를 “상위”로 만드는 첫 번째 축은 프로젝트 스케일 대응이다. 단일 보드 중심 사고에서 벗어나, 워크스페이스(프로젝트) 단위로 보드·폴더·태그·팔레트·노트를 묶어 관리하고, 하나의 워크스페이스 안에서 다수 보드를 탭으로 전환하며, 보드 간 레퍼런스를 “링크(단일 원본을 여러 보드에서 재사용)”로 취급해 중복 저장과 관리 비용을 줄인다. 워크스페이스는 파일 하나로 이식 가능한 패키지(.rnxpack)로 정의하되, 원본 이미지를 “경로 참조 모드(가볍고 빠름)”와 “내장 보관 모드(이동·백업에 강함)” 중 선택 가능하게 하여, 사용자가 성능과 휴대성을 스스로 최적화하게 한다.

두 번째 축은 수집 비용 절감이다. 드래그 앤 드롭과 클립보드 붙여넣기 같은 기본 수집 외에, (서버 없이) 브라우저 확장과 로컬 커스텀 프로토콜을 이용해 “현재 탭의 이미지/선택 영역/URL 메타데이터”를 즉시 RefNexus로 전송하는 단방향 파이프를 제공하고, 폴더 감시(Watch Folder)로 지정 경로에 새 이미지가 들어오면 자동으로 인박스 보드에 적재되게 한다. 이때 출처 URL, 캡처 시각, 저작권/라이선스 메모, 키워드(사용자 입력 또는 자동 추정)를 함께 저장하여, 레퍼런스를 “그림 조각”이 아니라 “근거가 붙은 자료”로 승격시키는 것을 목표로 둔다.

세 번째 축은 온디바이스 인덱싱과 검색이다. PureRef가 “눈으로 훑는 보드”에 강점이 있다면, RefNexus는 “찾아내는 보드”를 추가한다. 로컬에서 이미지 해시와 시각 임베딩을 생성해 중복(완전 동일)과 준중복(리사이즈·크롭·색감만 다른 사실상 동일 레퍼런스)을 자동 군집화하고, 텍스트-이미지 연관 검색을 온디바이스 모델로 수행해 “빨간 우산이 있는 거리 야경” 같은 질의를 로컬 레퍼런스에서 바로 검색하게 한다. 여기에 색상 팔레트 추출, 명도(밸류) 맵 자동 생성, 엣지(윤곽) 프리뷰, 평균 채도/대비 지표를 함께 제공하여, 미감 분석을 위해 외부 툴을 오가는 빈도를 줄인다.

네 번째 축은 배치·정리의 자동화와 표현력 확장이다. PureRef의 정렬·정돈 도구를 계승하되, RefNexus는 “스냅+제약조건”을 도입해 그리드/가이드/자석선에 의한 정렬을 기본화하고, 그룹에 잠금·정렬 기준·여백 규칙을 부여하여 대형 보드에서도 레이아웃이 쉽게 붕괴하지 않게 한다. 노트는 단순 메모가 아니라 체크리스트·링크·스타일을 포함하는 리치 텍스트로 제공하고, 이미지 위에 벡터 도형·화살표·캘리퍼(거리/비율 측정)·색상 샘플링을 얹는 “주석 레이어”를 별도로 두어, 레퍼런스 해석 과정 자체가 보드 안에 남도록 설계한다. PureRef 2.0+에서 노트가 리치 텍스트를 지원하는 흐름을 기준선으로 삼되, RefNexus는 주석 레이어의 분리와 잠금으로 “참조 이미지”와 “해석/지시”를 구조적으로 분리한다. ([PureRef][3])

다섯 번째 축은 안정성과 성능이다. RefNexus는 무한 캔버스의 대형화가 곧 성능 문제로 연결된다는 점을 전제로, 타일 기반 렌더링과 지연 로딩(뷰포트 밖의 이미지는 프리뷰만 유지), GPU 가속 디코딩, 썸네일/미리보기 캐시, 증분 저장을 기본으로 한다. 저장 포맷(.rnxpack)은 “매니페스트(JSON) + 썸네일 + 원본(선택) + 인덱스(SQLite)”로 구성하여, 보드 로딩은 빠르게, 원본 접근은 필요할 때만 수행되게 한다. 자동 저장은 기본 켜짐으로 두고, 시간 간격 스냅샷과 사용자가 찍는 스냅샷을 모두 지원하여 “되돌리기”가 파일 복구가 아니라 기능으로 작동하게 한다.

여섯 번째 축은 내보내기와 공유의 품질이다. PureRef가 캔버스 영역을 단일 이미지로 내보내는 기능을 제공하는 흐름을 계승하되, RefNexus는 인쇄·검토를 위해 PDF 내보내기(페이지 분할, 여백, 제목/메타데이터 포함), 레이어를 유지한 고해상도 이미지 내보내기, 그리고 “정적 뷰어 패키지(HTML+에셋)” 생성을 지원한다. 이는 서버 없이도 파일만 전달하면 동일한 보드를 다른 사람이 열람할 수 있게 하며, 팀 협업은 “클라우드 협업”이 아니라 “파일 기반 전달/버전 관리(Git LFS나 NAS 등 사용자의 기존 인프라 활용)”로 해결하는 방향을 취한다.

제품의 수익화는 서버가 없다는 조건과 모순되지 않게 “오프라인 라이선스 파일”로 구성한다. 개인용은 기능 제한형 무료 또는 기부형으로 진입 장벽을 낮추고, 프로 버전은 온디바이스 검색(임베딩), 중복 군집화, PDF/정적 뷰어 내보내기, 스냅샷 타임라인, 고급 제약 레이아웃 같은 고부가 기능을 포함한 영구 라이선스+유지보수(업데이트) 옵션으로 설계한다. 결과적으로 RefNexus는 PureRef가 제공하는 핵심 가치인 “항상 위, 최소 UI, 빠른 수집과 배치”를 보존하면서도, 규모 확장(프로젝트/라이브러리), 회수(검색/인덱스), 재현(스냅샷/패키지), 근거(출처/메타데이터)라는 네 축을 추가하여, 동일한 분야에서 한 단계 위의 작업 환경을 제공하는 상위 호환 앱으로 완결된다. ([PureRef][1])
